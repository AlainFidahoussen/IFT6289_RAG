## What did not work

- **Multimodal OCR prompt** — changed DeepSeek prompt to "Extract all text. For each figure or
  chart, describe its content in detail as part of the text flow." Did not improve NDCG@10;
  reverted to the default `<|grounding|>Convert the document to markdown.` prompt.
- **DeepSeek markdown cleaner** — stripped image placeholders (![](images/*.jpg)) and HTML table
  tags before embedding. No measurable impact on NDCG@10; removed.
- **DeepSeek-OCR-2 as text extractor** — overall NDCG@10 avg 55.82 (no rerank) and 71.02
  (zerank-2) vs NeMo 57.97 / 75.09. Gap widens after reranking (−2.15 → −4.07 pts).
  OCR output is noisier and longer than NeMo; does not help retrieval quality on these subsets.

## Mitigating DeepSeek-OCR-2's blindness to figures/charts

DeepSeek outputs image placeholders for visual content, so figures are invisible to the text stream.
The generator can compensate at answer-generation time:

**Primary (already in pipeline design)**
- Send the top-5 retrieved page *images* (from ColEmbed) alongside the top-5 markdown texts to the
  generator VLM. The model sees the actual figures directly — no OCR needed for visual content.
  This is the ViDoRe v3 hybrid protocol and is already planned.

**Alternatives if the hybrid context is insufficient**
- Figure captioning: DeepSeek already extracts figure images to disk (images/0.jpg, etc.) alongside
  the markdown. Run a captioning model (BLIP-2, LLaVA, or the generator VLM itself) on those
  saved images and replace the placeholders with the generated captions before embedding. Targeted
  and avoids re-running OCR.
- Multimodal OCR prompt: ask DeepSeek (or another model) to describe figures inline as part of the
  markdown. We tried this (prompt: "describe each figure in detail as part of the text flow") but
  reverted because it did not improve NDCG@10. May still help answer accuracy — worth revisiting
  after end-to-end QA is implemented.
